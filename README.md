# EvsConsole

Консольная утилита для конвертации аудиофайлов между форматом **EVS** (Enhanced Voice Services, 3GPP TS 26.443) и **WAV**.

Поддерживает как декодирование (EVS → WAV), так и кодирование (WAV → EVS).

## Возможности

- **Декодирование EVS → WAV**:
  - Автоопределение формата — EVS-ToC, MIME Storage, ITU G.192
  - Конвертация в WAV — PCM 16-bit mono, частота 8/16/32/48 кГц
  - Режим Whisper — выход 16 кГц, оптимальный для транскрибирования
  - Пакетная обработка — конвертация всех файлов в каталоге
  - Режим Probe — анализ структуры файла без декодирования

- **Кодирование WAV → EVS**:
  - Вход: WAV PCM 16-bit (mono или stereo, автоматический downmix в mono)
  - Частота дискретизации: 8000, 16000, 32000 или 48000 Гц
  - Битрейт: от 7.2 кбит/с до 128 кбит/с
  - Выходной формат: G.192 или MIME Storage
  - Пакетная обработка директорий с WAV файлами

## Требования

- .NET 10+ (или текущий .NET SDK)
- `evs_dec.exe` — декодер из 3GPP TS 26.443 (отсюда https://www.3gpp.org/ftp/Specs/archive/26_series/26.443/)

## Использование

### Декодирование EVS → WAV

```bash
# Автоопределение формата, выход в WAV рядом с исходным файлом
EvsConsole data_file.bin

# Указать выходной файл и частоту дискретизации
EvsConsole data_file.bin output.wav -s 16
```

### Кодирование WAV → EVS

```bash
# Кодирование WAV в EVS (битрейт по умолчанию: 24.4 kbps, формат: G.192)
EvsConsole --encode audio.wav

# Указать битрейт и выходной файл
EvsConsole --encode audio.wav output.evs -b 32000

# Кодирование в MIME формате (рекомендуется для компактности)
EvsConsole --encode audio.wav output.evs --format mime

# Пакетное кодирование всех WAV файлов в директории
EvsConsole --encode recordings/ output_evs/
```

### Подготовка WAV через ffmpeg

Для кодирования в EVS рекомендуется использовать следующие параметры ffmpeg:

```bash
# 16 кГц mono PCM — оптимально для EVS и Whisper
ffmpeg -i input.mp3 -ar 16000 -c:a pcm_s16le -ac 1 output.wav

# 8 кГц mono (минимальный размер)
ffmpeg -i input.mp3 -ar 8000 -c:a pcm_s16le -ac 1 output.wav

# 48 кГц mono (полная полоса)
ffmpeg -i input.mp3 -ar 48000 -c:a pcm_s16le -ac 1 output.wav
```

Параметры:
- `-ar 16000` (или 8000/32000/48000) — частота дискретизации
- `-c:a pcm_s16le` — PCM 16-bit little-endian
- `-ac 1` — моно (энкодер также автоматически делает downmix)

### Конвертация для Whisper

```bash
# 16 кГц mono PCM — оптимальный формат для OpenAI Whisper
EvsConsole data_file.bin --whisper

# Пакетная конвертация для Whisper
EvsConsole recordings/ output/ --whisper
```

### Пакетная конвертация

```bash
# Конвертировать все EVS файлы в каталоге
EvsConsole recordings/ output/

# Конвертировать все WAV файлы в EVS
EvsConsole --encode wav_recordings/ evs_output/
```

### Анализ файла (probe)

```bash
EvsConsole --probe data_file.bin
```

Вывод probe — hex-дамп заголовка, определённый формат, список фреймов с битрейтами и длительность.

### Все опции

| Опция | Описание |
|---|---|
| **Декодирование** | |
| `-s`, `--samplerate <кГц>`  | Частота дискретизации выхода: 8, 16, 32 или 48 (по умолчанию: 8) |
| `-w`, `--whisper`           | Формат для Whisper: 16 кГц, моно, PCM 16-bit |
| `-d`, `--decoder <путь>`    | Путь к `evs_dec.exe` (по умолчанию: ищет рядом с приложением) |
| **Кодирование** | |
| `-c`, `--encode [bitrate]`  | Режим кодирования WAV → EVS |
| `-b`, `--bitrate <bps>`     | Битрейт: 7200-128000 bps (по умолчанию: 24400) |
| `--format <g192\|mime>` | Выходной формат: G.192 или MIME (по умолчанию: g192) |
| `-e`, `--encoder <путь>`    | Путь к `evs_cod.exe` |
| **Общие** | |
| `-p`, `--probe`             | Анализ структуры файла без декодирования |
| `-q`, `--quiet`             | Минимальный вывод (только ошибки и результат) |
| `--keep-temp`               | Не удалять временные файлы (для отладки) |
| `-h`, `--help`              | Справка |

## Поддерживаемые форматы

Формат определяется автоматически по содержимому файла.

### EVS-ToC (основной сценарий)

Последовательность фреймов `[ToC][data][ToC][data]...`,
где ToC-байт содержит тип кодека и режим битрейта.

Структура ToC-байта:

```
Бит:  |  7  |  6  |  5  |  4  | 3  2  1  0 |
      |  H  |  F  |  E  |  —  |    mode    |
```

- **H, F** — служебные биты из RTP-заголовка (очищаются при конвертации)
- **E** — тип кодека: 0 = EVS Primary, 1 = AMR-WB IO
- **mode** — индекс битрейта (см. таблицы ниже)

Утилита добавляет MIME-заголовок `#!EVS_MC1.0\n` + 4 байта каналов и передаёт в `evs_dec.exe -mime`.

### MIME Storage Format

Файл начинается с magic word `#!EVS_MC1.0` или `#!AMR-WB`. Передаётся напрямую в `evs_dec.exe -mime`.

### ITU G.192

Формат, описанный в рекомендации ITU-T G.192. Каждый фрейм (20 мс):

```
Word16  SyncWord   (0x6B21 — OK, 0x6B20 — bad frame)
Word16  DataLen    (количество бит данных)
Word16  1st Bit    (0x007F = 0, 0x0081 = 1)
Word16  2nd Bit
...
Word16  Nth Bit
```

Передаётся напрямую в `evs_dec.exe` (формат по умолчанию).

## Принцип работы

### Декодирование (EVS → WAV)

```
┌──────────────┐     ┌─────────────────┐     ┌──────────────┐     ┌──────────┐
│  Входной     │     │ Определение     │     │ Подготовка   │     │ evs_dec  │
│  файл (.bin) │───▶│ формата         │────▶│ (ToC→MIME)   │───▶│ .exe     │
│              │     │ (auto-detect)   │     │              │     │          │
└──────────────┘     └─────────────────┘     └──────────────┘     └────┬─────┘
                                                                       │ PCM
                                                                       ▼
                                                                 ┌──────────┐
                                                                 │ WAV      │
                                                                 │ header + │
                                                                 │ PCM data │
                                                                 └──────────┘
```

1. Чтение бинарного файла
2. Автоопределение формата по сигнатурам
3. Для EVS-ToC: парсинг фреймов, очистка H/F-бит, добавление MIME-заголовка
4. Вызов `evs_dec.exe` с временным файлом → получение raw PCM
5. Добавление WAV-заголовка (44 байта RIFF/WAVE) → запись результата

### Кодирование (WAV → EVS)

```
┌──────────────┐     ┌─────────────────┐     ┌──────────────┐     ┌──────────┐
│  Входной     │     │ Парсинг         │     │  Downmix     │     │ evs_enc/ │
│  WAV файл    │───▶│ WAV заголовка   │────▶│  (если stereo)│───▶│ evs_cod  │
│              │     │ (PCM, rate, ch) │     │              │     │ .exe     │
└──────────────┘     └─────────────────┘     └──────────────┘     └────┬─────┘
                                                                       │ EVS
                                                                       ▼
                                                                 ┌──────────┐
                                                                 │ EVS      │
                                                                 │ bitstream│
                                                                 │ (.192/.evs)
                                                                 └──────────┘
```

1. Чтение WAV файла, проверка формата (PCM 16-bit)
2. Извлечение PCM данных, sample rate, channel count из заголовка
3. Downmix stereo в mono (усреднение каналов)
4. Вызов `evs_cod.exe` с PCM данными → получение EVS bitstream
5. Запись EVS файла в формате G.192 или MIME
